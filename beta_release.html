DeepImageJ Beta Release

Your feedback is crucial for to improve, so please, if you experience any problem, miss anything or simply, you would like to comment on the beta version of deepImageJ, do not hesitate to contact us.

New functionalities of deepImageJ 2.1.0 Beta Release

  • Compatibility with the Bioimage Model Zooü¶í: deepImageJ is now one of the consumers of the trained models in Bioimage.IO. Hence, the configuration file config.xml was updated to the new standardized configuration file model.yaml
  • From Image2image in 2D to image2any: this beta release runs models defined for 3D image processing, image classification or object detection through region proposal networks. It deals with miscellaneous output formats and shapes.
  • GPU support.
  • New formats for image pre and post-processing: ImageJ macros (.txt/.ijm) and Java code&classes (.jar/.class) are supported and can be combined.
  • Compatibility with PyTorch: All PyTorch models stored as TorchScript can be for the first time loaded into Fiji through deepImageJ.

Installation guidelines

For the moment, it is recommended to use a clean Fiji/ImageJ set up. A prebuilt Fiji with deepImageJ preinstalled for Linux is provided here.

Find further details about installation and dependencies at deepImageJ's Wiki

  • Download the dependencies and the plugin given in the following links.
    • Dependencies (~380 Mb)
    • DeepImageJ 2.1.3
  • Place DeepImageJ_-2.1.0-SNAPSHOT.jar inside the 'plugins' folder of the 'ImageJ2/Fiji' directory.

  • Unzip the dependencies.zip file. Copy all the .jar files in the 'jars' folder inside the 'ImageJ2/Fiji' directory. If we already have other versions of the dependencies inside the 'jars' folder, the plugin might not function correctly. For a quick test, if possible, we suggest to remove the duplicated dependencies and leave the ones we provide.

  • PyTorch models in Windows: If you are using Windows, please make sure that you have Visual Studio installed:
    • Install Visual Studio 2019 redistributable.
    • Install Visual Studio 2019.
  • GPU connection.

    Detailed information for the GPU installation is given here.

    Requirements:
    • TensorFlow: CUDA 9.0 (TensorFlow 1.12) / CUDA 10.0 (TensorFlow 1.15) and corresponding CUDnn and drivers.
    • PyTorch: CUDA 10.2 is accepted, BUT having other CUDA versions installed might be source of conflicts.
      • In Windows, if any version other than CUDA 10.2 is installed, the plugin will fail to load the model (link to the bug issue in GitHub).
      • If deepImageJ is not able to load a PyTorch model using the GPU, remove CUDA_PATH (if exists) from your environment variables and make sure that your PATH does not include any directory or file with the words ‚ÄúNvidia‚Äù or ‚ÄúCUDA‚Äù. For that go to 'Edit the system environment variables' or 'Edit environment variables for your account', then click on 'Environment variables' and finally check the 'Path' and 'CUDA_PATH' variables (note that Windows is not case sensitive so they might be written and 'PATH' or 'path').

Models for testing

While compatible with the Bioimage Model Zooü¶í, these models are not synchronized yet as they are only compatible with the Beta release of deepImageJ.

To install them in ImageJ, create a folder called models at /ImageJ/ or /Fiji/. Download any of the following models. Unzip its content and place it in the new folder /ImageJ/models.

For example, if you download the model MaskR-CNN, you will get the zip file MaskRCNN.bioimage.io.model.zip. Once unzipped, it should look in ImageJ as /ImageJ/models/MaskRCNN.bioimage.io.model, and inside it, the contents (exampleImage.tif, model.yaml, General.jar, tensorflow_saved_model_bundle.zip, ...).
