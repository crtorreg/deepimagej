DeepImageJ Beta Release

Your feedback is crucial for to improve, so please, if you experience any problem, miss anything or simply, you would like to comment on the beta version of deepImageJ, do not hesitate to contact us.

New functionalities of deepImageJ Beta Release

  • Compatibility with the Bioimage Model Zooü¶í: deepImageJ is now one of the consumers of the trained models in Bioimage.IO. Hence, the configuration file config.xml was updated to the new standardized configuration file model.yaml
  • From Image2image in 2D to image2any: this beta release runs models defined for 3D image processing, image classification or object detection through region proposal networks. It deals with miscellaneous output formats and shapes.
  • GPU support.
  • New formats for image pre and post-processing: ImageJ macros (.txt/.ijm) and Java code&classes (.jar/.class) are supported and can be combined.
  • Compatibility with PyTorch: All PyTorch models stored as TorchScript can be for the first time loaded into Fiji through deepImageJ.

Quick installation guidelines

It is recommended to use a clean Fiji/ImageJ set up avoid version-conflict with previously installed dependencies.

  • Download the dependencies and the plugin given in the following links.
    • Dependencies 2.1.6 (~380 Mb)
    • DeepImageJ 2.1.6
  • Place DeepImageJ_-2.1.4-SNAPSHOT.jar inside the 'plugins' folder of the 'ImageJ2/Fiji' directory.

  • Unzip the dependencies_214.zip file. Copy all the .jar files in the 'jars' folder inside the 'ImageJ2/Fiji' directory. If we already have other versions of the dependencies inside the 'jars' folder, the plugin might not function correctly. For a quick test, if possible, we suggest to remove the duplicated dependencies and leave the ones we provide.

  • PyTorch models in Windows: If you are using Windows, please make sure that you have Visual Studio installed:
    • Install Visual Studio 2019 redistributable.
    • Install Visual Studio 2019.

GPU connection (optional)

Requirements:
  • TensorFlow: CUDA 9.0 (TensorFlow 1.12) / CUDA 10.0 (TensorFlow 1.15) and corresponding CUDnn and drivers.
  • PyTorch: CUDA 10.2 is accepted, BUT having other CUDA versions installed might be source of conflicts.
    • In Windows, if any version other than CUDA 10.2 is installed, the plugin will fail to load the model (link to the bug issue in GitHub).
    • If deepImageJ is not able to load a PyTorch model using the GPU, remove CUDA_PATH (if exists) from your environment variables and make sure that your PATH does not include any directory or file with the words ‚ÄúNvidia‚Äù or ‚ÄúCUDA‚Äù. For that go to 'Edit the system environment variables' or 'Edit environment variables for your account', then click on 'Environment variables' and finally check the 'Path' and 'CUDA_PATH' variables (note that Windows is not case sensitive so they might be written and 'PATH' or 'path').

Play with deepImageJ (suggested steps)

The easiest way to start is as follows:
  • Click on DeepImageJ Install, and install one of the suggested models compatible with deepImageJ from the list. This list refers to all the models in the BioImage Model Zoo. The ones compatible whit deepImageJ are marked in green. There's no need to restart Fiji once the model is installed.
  • Once the model is installed, you can find a demo image (tif) in the folder created at Fiji/models.
  • Open the image in ImageJ and process it with deepImageJ (DeepImageJ Run).
And if you wan to create your own bundled model? If you are familiarized with deep learning models and architectures, this shouldn't be too difficult.
  • Download one of the models from the BioImage Model Zoo. You will (re)build the bundled model yourself with DeepImageJ Build Bundled Model.
  • This step assumes that you are using a "raw Python model", go to the model folder (the one you just downloaded) and extract the zip called "tensorflow_saved_model_bundle.zip". This will result in a "saved_model.pb" and a "variables" folder.
  • Run DeepImageJ Build Bundled Model.
  • Drag&drop the folder containing saved_model.pb and a variables folder (it doesn't matter if there are other files).
  • Follow the steps of the user guide. As you are not the model creator you may need to check some parameters in the file called model.yaml, in the model folder.
  • The pre- and post-processing files, and example data are also in the model folder.
DeepImageJ Run is macro recordable!
  • Start the macro recorder.
  • Use DeepImageJ Run on an image.
  • Copy the code line given by the macro recorder.
  • Create a new ImageJ macro, paste the line and run it.
